\section{Abstract}

AISSMS IOIT ACM Student Chapter is organizing a on-site Capture The Flag competition on classical information security challenges plus emerging AI security threats. We are seeking partner ACM Student Chapters across the region to collaborate and scale this into a premier regional cybersecurity event.

The event targets students, early career professionals, and security enthusiasts, offering hands-on workshops and a competitive CTF component (12 hours of competition, structured as either a single intensive day or split across two days).

\section{Event Overview}

\begin{itemize}
    \item \textbf{Organizing Chapter:} AISSMS IOIT ACM Student Chapter
    \item \textbf{Format:} On-site CTF (jeopardy-style) with specialized AI/ML security tracks
    \item \textbf{Capacity:} $\sim$120 onsite participants
    \item \textbf{Tentative Window:} March–April 2026 (flexible)
    \item \textbf{Event Vision:} Regional-scale collaborative CTF event
\end{itemize}

\section{Event Tracks \& Challenge Scope}

Each track includes easy $\to$ medium $\to$ hard challenges. Total: \textbf{35–50 challenges} designed to accommodate various skill levels and ensure competitive scoring distribution.

\subsection{Core Security Tracks}

\begin{itemize}
    \item \textbf{Web Security} — Web vulnerabilities, authentication bypass, logic flaws, session management
    \item \textbf{OSINT} — Open-source intelligence, footprinting, metadata analysis, social engineering puzzles
    \item \textbf{Mobile Security} — Android/iOS app reversing, insecure storage, inter-process communication
    \item \textbf{Digital Forensics} — PCAP analysis, log examination, timeline reconstruction, evidence recovery
    \item \textbf{Network Security} — Protocol analysis, traffic inspection, service exploitation
    \item \textbf{Cryptography} — Classical and modern cryptography, encoding puzzles, implementation flaws
    \item \textbf{Reverse Engineering} — Binary analysis, obfuscation techniques, packer analysis
    \item \textbf{Miscellaneous} — Steganography, logic puzzles, hardware simulation
\end{itemize}

\subsection{AI/ML Security Tracks (Specialized)}

Given the emerging importance of AI security, we're dedicating expanded coverage:

\begin{itemize}
    \item \textbf{Prompt Injection \& Manipulation} — Direct and indirect prompt injection, jailbreaking techniques, context manipulation
    \item \textbf{Model Context Protocol (MCP) Exploitation} — MCP security vulnerabilities, tool abuse, context poisoning
    \item \textbf{Adversarial Machine Learning} — Adversarial examples, model evasion, data poisoning attacks
    \item \textbf{AI Model Hardening} — Input sanitization challenges, output filtering, safety boundary testing
    \item \textbf{LLM Logic \& Reasoning Exploits} — Chain-of-thought manipulation, reasoning bypass, hallucination exploitation
    \item \textbf{AI Application Security} — RAG system vulnerabilities, agent security, AI API abuse
\end{itemize}

\noindent \textbf{Flag Format:} \texttt{ACMCTF\{...\}} (consistent across all challenges)

\noindent \textbf{Suggested Challenge Distribution} (for $\sim$40 challenges):\\
Web 5, OSINT 4, Mobile 3, Forensics 5, Network 4, Crypto 4, RE 5, Misc 3, Prompt Injection 2, MCP Security 2, Adversarial ML 1, Model Hardening 1, LLM Logic 1

\section{Competition Format Options}

\subsection{Option A — Single Day Intensive (Recommended for focused execution)}
\begin{itemize}
    \item \textbf{12 continuous hours} of competition (onsite)
    \item Morning: Opening ceremony, sponsor keynotes, quick technical briefings
    \item Midday-Evening: Full competition window
    \item Ideal for: Simplified logistics, single-day venue booking, concentrated energy
\end{itemize}

\subsection{Option B — Two-Day Educational + Competition (Recommended for learning focus)}
\begin{itemize}
    \item \textbf{Day 1:} Workshops on security fundamentals and AI/ML security, warmup challenges, 6-8 hour qualifier round (open to all registered participants)
    \item \textbf{Day 2:} Final 6-8 hour main event for top N teams (determined by Day 1 performance), closing ceremony, prize distribution, networking session
\end{itemize}

\noindent \textbf{Our Recommendation:} Option B maximizes educational value and allows workshops to prepare participants for specialized AI security challenges, while creating a more engaging multi-day experience.

\section{Rules \& Code of Conduct}

\subsection{Competition Rules}
\begin{itemize}
    \item \textbf{Strictly on-site} — No remote participation (ensures fair play and community building)
    \item \textbf{Team Structure} — Solo participants and teams of 1-4 members allowed
    \item \textbf{Fair Play} — No flag sharing, inter-team collaboration, or external assistance
    \item \textbf{Flag Format} — \texttt{ACMCTF\{...\}} must be maintained
    \item \textbf{Scoring System} — Point-based with dynamic scoring (first solves earn bonus points); time used as tiebreaker
    \item \textbf{Writeups} — Participants may be asked to submit brief writeups for solved challenges (post-event)
\end{itemize}

\section{Technical Infrastructure (Minimum Requirements)}

\subsection{Core Platform}

\textbf{Scoring Platform}
\begin{itemize}
    \item CTFd or equivalent hosted on resilient infrastructure from a CTF Partner
\end{itemize}

\textit{The CTF Partner will provide following deleveriables:}
\begin{itemize}
    \item HTTPS with valid certificates
    \item Rate limiting and DDoS protection
    \item Backup systems and redundancy
    \item Containerized environments (Docker/LXC/KVM) for each challenge
    \item Orchestration for easy deployment and rebuild
    \item Auto-cleanup policies to prevent resource exhaustion
    \item Isolated networks per challenge category
    \item NAT and port forwarding only for intended services
    \item Internal DDoS prevention and monitoring
    \item Firewall rules isolating challenge networks
    \item Network traffic monitoring and logging
    \item Load testing in staging environment prior to event
    \item Backup and rollback images for all challenges
    \item Real-time monitoring dashboards for uptime and performance
    \item Alert systems for infrastructure issues
\end{itemize}
